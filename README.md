# Задание 1
## Задача 10
### Текст задачи
Составить список всех трехзначных чисел, в десятичной записи которых нет
одинаковых цифр (операции деления и нахождения остатка от деления не
использовать).
### Алгоритм решения
С помощью набора фактов digit от 0 до 9 программа определяет доступные элементы для построения чисел.
Рекурсивная функция not_member:
Базовый случай: Если список пуст, любой элемент считается отсутствующим в нем.
Рекурсивный шаг: Проверяет, не равен ли искомый элемент голове списка, и рекурсивно переходит к проверке хвоста.
Рекурсивная функция all_different:
Базовый случай: Пустой список признается списком с уникальными элементами.
Рекурсивный шаг: Берет первый элемент и с помощью not_member проверяет его отсутствие в остальной части списка, затем рекурсивно вызывает себя для хвоста.Процесс генерации в main:Используется встроенный предикат findall для сбора всех комбинаций цифр S, D и E.
Устанавливаются ограничения: S (сотни) не может быть нулем, а все три цифры в списке должны удовлетворять условию all_different.
Функция вывода print_all:
Рекурсивно обходит полученный список троек цифр.На каждом шаге выводит три цифры подряд, имитируя запись целого числа, и пробел.

# Задание 2
## Задача 10
### Текст задачи
Найти разность максимального и минимального элементов списка
### Алгоритм решения
Запрос данных: Программа запрашивает у пользователя ввод списка чисел через read(List).
Проверка ввода: С помощью is_list и сравнения с пустым списком [] проверяется корректность данных. Если список пуст, выводится сообщение об ошибке.
Рекурсивная функция find_max:
Базовый случай: Если в списке один элемент, он является максимальным.
Рекурсивный шаг: Сравнивает голову списка с результатом рекурсивного поиска максимума в хвосте, выбирая наибольшее значение.
Рекурсивная функция find_min:
Аналогична поиску максимума, но на этапе сравнения выбирает наименьшее из двух значений (головы или минимума хвоста).
Вычисления в main:
После получения значений MaxVal и MinVal программа использует оператор is для вычисления их разности.
Результаты (максимум, минимум и их разность) выводятся в консоль.

# Задание 3
## Задача 10
### Текст задачи
Определим множество как список без повторяющихся элементов. Для двух
заданных множеств определить, является ли одно из них подмножеством
другого.
### Алгоритм решения
Запрос данных: Программа последовательно запрашивает ввод двух множеств (списков) у пользователя.
Рекурсивная функция element_of:
Базовый случай: Если искомый элемент совпадает с головой списка, он считается найденным.
Рекурсивный шаг: Если совпадения нет, функция рекурсивно ищет элемент в хвосте списка.
Рекурсивная функция is_subset:
Базовый случай: Пустой список признается подмножеством любого другого множества.
Рекурсивный шаг: Проверяет наличие первого элемента (головы) первого списка во втором списке через element_of и рекурсивно продолжает проверку для оставшейся части.
Логика сравнения в main:
Используется конструкция if-then-else (-> ;).
Сначала проверяется, является ли первый список подмножеством второго. Если нет, проверка запускается в обратную сторону.
В зависимости от результата выводится соответствующее текстовое сообщение.

# Задание 4
## Задача 10
### Текст задачи
Пять детей - Алик, Боря, Витя, Лена и Даша - приехали в лагерь
из 5 разных городов: Харькова, Умани, Полтавы, Славянска и Краматорска. Есть
4 высказывания:
1) Если Алик не из Умани, то Боря из Краматорска.
2) Или Боря, или Витя приехали из Харькова.
3) Если Витя не из Славянска, то Лена приехала из Харькова.
4) Или Даша приехала из Умани, или Лена из Краматорска.
Кто откуда приехал?
### Алгоритм решения
Генерация вариантов в solve:
Определяется список доступных городов.
С помощью member каждому из пяти детей (Алик, Боря, Витя, Лена, Даша) назначается город.
Проверка уникальности: Вызывается рекурсивная функция all_different, которая гарантирует, что ни один город не назначен двум детям одновременно.
Проверка логических условий:
Условие 1: Реализовано как логическое ИЛИ между тем, что Алик из Умани, и тем, что Боря из Краматорска.
Условие 2 (XOR): Проверяет, что либо Боря из Харькова, либо Витя из Харькова (но не оба сразу).
Условие 3: Реализовано через проверку: либо Витя из Славянска, либо Лена из Харькова.
Условие 4 (XOR): Проверяет исключающее ИЛИ для Даши в Умани и Лены в Краматорске.
Процесс поиска в main:
Prolog использует механизм бэктрекинга для перебора всех комбинаций, пока не будет найдено распределение, удовлетворяющее всем четырем правилам.
Найденные соответствия «имя — город» выводятся на экран.
